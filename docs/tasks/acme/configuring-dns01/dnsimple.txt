=========================
DNSimple
=========================

This provider uses a Kubernetes ``Secret`` resource to access
the DNSimple API Token. In this example the ``Secret`` must be
named ``dnsimple-oauth-token`` with the token stored in the
data key ``oauth-token``


Please review the `DNSimple documentation <https://support.dnsimple.com/articles/api-access-token/>`_
for help on issuing a DNSimple API token.



 .. code-block:: yaml
   dnsimple:
    oauthTokenSecretRef:
      name: dnsimple-oauth-token
      key: oauth-token

If your Kubernetes cluster is having issues resolving
CAA queries for your DNSimple domain, you may add the DNSimple
nameservers for resolution or another set of nameservers.

 .. code-block:: sh
  --dns01-recursive-nameservers=ns1.dnsimple.com:53,ns2.dnsimple.com:53,ns3.dnsimple.com:53,ns4.dnsimple.com:53,
  # OR
  --dns01-recursive-nameservers=8.8.8.8:53,1.1.1.1:53
